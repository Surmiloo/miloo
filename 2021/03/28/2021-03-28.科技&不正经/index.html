1 停止使用密码遮挡

2014年，一项学术研究调查了密码屏蔽对用户的影响。

密码屏蔽是一种非常常见的设计模式。它涉及一个文本字段，该文本字段可以接受任何字符，但是不会向用户显示输入的字符。相反，它显示了一个要点。为了提高安全性，在几乎所有需要用户登录的地方，都可以找到它的示例。

该研究就密码屏蔽提出了一下问题：

密码屏蔽存在许多可用性问题：

  • 无法看到输入了哪些字符
  • 无法检查输入
  • 无法纠正错误

对于所有这些给用户带来的不便，实际上根本没有安全方面的好处：

  • 当您输入密码时，某人多久看一次您的肩膀？
  • 输入没有任何保护，只是一种视觉表示
  • 密码屏蔽无助于防止来自关键记录器或恶意软件的攻击

有专家指出：密码屏蔽是一种遗留模式，无需任何考虑就可以继续采用。但是它所做的只是给人们带来不便。

80％的人不希望密码以明文形式显示，他们指出：密码屏蔽可以保护自己免受他人的侵害。这让自己感觉到已经有了某种形式的保护。（PS：这或许是弱口令存在的另一种原因）

结论：明文密码确实可以提高可用性，我们应当将它作为一种可选方式，至于默认设置使用密码屏蔽与否这将是另外一个问题。

Tip:时至今日，已经存在一部分网页提供了可选的密码屏蔽服务。

2 “摸鱼学导论”引发的悖论

摸鱼学导论

清华大学最近开设了一门课程《摸鱼学导论》，普天之下摸鱼人的心都是共通的，目前选修人员已经爆满。

这门课程不点名、不考试、没有作业、也不进行期末考核，有人问期末成绩怎么计算，老师答：“如果你深入贯彻落实摸鱼精神，你不问我这个问题你就是满分了。”

这门课程一经诞生就产生了一个极大的悖论，摸鱼课以鼓励和教导别人摸鱼为主，那么在课程上摸鱼实际上就是在努力内卷，而在课程中努力学习那就是摸这门课程的鱼。所以问题来了，你在这门课程中的行为到底是摸还是卷？于是造成了摸卷二向性问题。

有人表示：这门课程的授课老师水平不高，如果老师水平高，就应该以身作则，率先摸鱼，把这门课程的教学任务给鸽了。

3 弓箭手悖论-你的箭是游出去的

弓箭手悖论

上世纪三十年代，人们在射箭运动中发现了一个神奇现象：普通人下意识地认为射箭时就应该把箭对准目标，而实际情况却是，弓箭手搭在弦上的箭是偏向一侧的，箭在射出后不但不会跑偏还能准确地命中目标，反倒将箭瞄准目标射击，就一定会打偏。这种奇怪的现象就被称为“弓箭手悖论”。

6 CSS 漏洞泄漏用户信息

CSS 漏洞泄漏用户信息

黑客利用CSS漏洞窃取用户信息，操作令人叹为观止。

黑客诱导用户访问一个恶意网页，里面嵌入 iframe 加载用户 facebook 主页。然后用一个单像素图片，逐一放在 iframe 的每个像素上面，再使用 mix-blend-mode 的 CSS 设置，根据渲染时间差异，算出原始像素的颜色，20秒可以拿到用户名。

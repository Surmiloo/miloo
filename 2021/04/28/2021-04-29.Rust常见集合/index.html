  2. 可增长、可修改、可拥有
  3. UTF-8
  • 其它类型的字符串：OsString、OsStr、CString、CStr
  1. 以 String 后缀结尾的可以获得所有权，以 Str 结尾的为借用或者变体

3 hashmap

3.1 HashMap<K, V>

  • 键值对的形式存储数据，一个键(Key)对应一个值(Value)
  • Hash 函数：决定如何在内存中存放 K 和 V
  • 适用场景：通过 K 来寻找数据，并非索引

3.2 创建 HashMap

  • HashMap 用的比较少，所以不在预导入模块
  • 标准库对 HashMap 支持比较少，没有内置的宏来创建 HashMap
  • 数据存储在 heap 上
  • HashMap 是同构的：一个 HashMap 中所有的 key 必须是同一种类型，value 必须是同一种类型

3.2.1 new 函数创建 HashMap

  • 创建空 HashMap: new() 函数
  • 添加数据: insert() 方法

3.2.2 collect 方法创建 HashMap

  • 在元素类型为 Tuple 的 Vector 上使用 collect 方法，可以组件一个 HashMap
  1. 要求 Tuple 有两个值： 一个作为 K, 一个作为 V
  2. collect方法可以把数据整合成很多中集合类型，包括 HashMap (返回值需要显式指明类型)

3.3 HashMap 和所有权

  • 对于实现了 Copy trait 的类型，值会被复制到 HashMap 中，如：i32
  • 对于拥有所有权的值，会被移动，所有权会转移给 HashMap，如：String
  • 在 HashMap 有效期间内，有效的值必须保持有效

3.4 遍历/访问 HashMap 中的值

  • get方法:
  1. 参数: K
  2. 返回: Option<&V>

3.5 更新 HashMap

  • HashMap 大小可变
  • 每个 K 同时只能对应一个 V
  • 更新 HashMap 中的数据：
  1. K 已经存在，对应一个 V: 1替换现有的V; 2保留现有的V,忽略新的V; 3合并现有的V和新的V
  2. K 不存在: 添加一对 K, V

3.6 哈希函数

HashMap 默认使用一种 “密码学安全的”（“cryptographically strong” ）哈希函数，它可以抵抗拒绝服务（Denial of Service, DoS）攻击。然而这并不是可用的最快的算法，不过为了更高的安全性值得付出一些性能的代价。如果性能监测显示此哈希函数非常慢，以致于你无法接受，你可以指定一个不同的 hasher 来切换为其它函数。hasher 是一个实现了 BuildHasher trait 的类型。第十章会讨论 trait 和如何实现它们。你并不需要从头开始实现你自己的 hasher；crates.io 有其他人分享的实现了许多常用哈希算法的 hasher 的库。
